# Skill: Test Driven Development

## 목적

Claude Code가 기능을 바로 구현하지 않고, 실패 테스트 → 최소 구현 → 리팩터링 순서로 작업하게 한다.

## 사용 시점

- 건강 데이터 처리
- 리포트 생성
- 챗봇 응답 로직
- 인증/권한
- Supabase DB 로직
- AI 출력 파싱
- n8n webhook 처리
- 결제/전환/CTA 핵심 흐름

## 절차

### 1. 테스트 대상 식별

먼저 다음을 찾는다.

- 관련 함수
- 관련 service
- 관련 API/action
- 관련 schema
- 관련 test file

없으면 가장 작은 테스트 파일을 만든다.

### 2. 실패 테스트 작성

구현 전에 테스트를 작성한다.

테스트는 다음을 포함한다.

- 성공 케이스
- 실패 케이스
- 권한/보안 케이스
- 빈 입력/잘못된 입력
- AI 출력 오류 케이스

### 3. 최소 구현

테스트를 통과할 만큼만 구현한다.
불필요한 추상화는 만들지 않는다.

### 4. 리팩터링

테스트가 통과한 뒤 구조를 개선한다.

리팩터링 기준:

- 중복 제거
- 함수 책임 분리
- 타입 명확화
- user-facing 문구 정리
- 에러 처리 정리

### 5. 검증

작업 종료 전 확인한다.

- 테스트 통과
- 타입 체크 통과
- lint 통과
- 의료적 위험 표현 없음
- 민감 정보 로그 없음

## 금지

- 테스트 없이 핵심 로직 구현
- 실제 LLM API를 단위 테스트에서 호출
- 실제 사용자 데이터를 테스트에 사용
- 실패 테스트 확인 없이 구현 완료 선언
- 큰 구조 변경을 테스트 없이 수행

## Claude Code 응답 형식

작업 후 다음 형식으로 보고한다.

```text

-

- 작성한 테스트:
- 실행한 명령:
- 결과:

주의할 점:
- 

다음 추천 작업:
- 
```
