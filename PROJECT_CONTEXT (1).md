## 0. 한 줄 정의
사용자의 상태(수면·스트레스 등)와 위치를 받아, 전국 치유의숲 38개 중 **"왜 이 숲인지"를 과학적 근거로 설명하는 처방**을 내리고, 방문 전후 효과를 측정하는 산림치유 추천·근거 엔진.

## 1. 대회 프레이밍 (이걸로 평가받는다)
- **추천이 메인**, 효과 측정(Evidence)이 추천의 신뢰 근거.
- 차별화 핵심 = **공공데이터 융합**: 치유의숲 좌표 + 청정넷 실시간 기상/미세먼지 + 국립산림과학원 피톤치드 연구.
- 발표 클라이맥스 = **유형별 정반대 추천**(인터뷰 근거). 같은 숲, 다른 사용자 → 다른 1등. (검증됨)
- 정직성 = 추정 지수는 "측정값 아님" 명시. 표본 작음·내부자 편향 인정 → 신뢰 해자.

## 2. 사용자 유형 (실제 인터뷰 2명 기반, 2026-06)
정반대 선호 관찰 → **단일 랭킹 불가, 유형 분기 필수**.
- **comfort(편안함형, 디폴트·일반 타겟)**: 거리가 결정의 전부. 잠/편안함 중심. AI·유튜브로 탐색. (여성/유방암경험자)
- **explorer(근거형)**: 거리 무관. 차별성·과학적 근거·특화 경험. (남성/내부자)
- 유형 감지: **사전설문 질문 1개**("거리가 중요 vs 특별한 경험이 중요"). 과설계 금지.
- 디폴트 comfort. explorer는 근거 레이어 정당화 보조.
- **킬러 아웃컴 = 수면.** 데모 페르소나 "잠 못 자던 사람이 숲에서 숙면" 라인.

## 3. 기술 스택

## 5. 처방 엔진 로직 (확정, v1 = 3축)
점수 축 3개(각 0~100) → 유형별 가중치 → 랭킹 → 처방전.
```
① 거리(접근성)   haversine (운영시 카카오 길찾기로 업글, 데모는 직선)
② 피톤치드 잠재력 phytoncide_index.py (보고서 근거)
③ 미세먼지 안전   청정넷 PM2.5 (낮을수록 가점, >35 회피)
```
※ ④ 프로그램 매칭은 제외(§4). 제거분(.15/.30)을 ①③에 재분배.
가중치(합=1.0):
- comfort  = {거리 .50, 피톤치드 .20, 미세먼지 .30}
- explorer = {거리 .10, 피톤치드 .60, 미세먼지 .30}

처방전 출력: 추천 숲 + 권장 시점(여름 정오 회피, 오전) + **왜 이 숲(근거 인용)** + 기대 효과 + 사후 측정(수면 중심: 전 수면점수·피로도 / 직후 회복감 / 3일후 수면점수).

검증됨(2026-06): comfort→가까운 숲, explorer→피톤치드 최고 편백. 같은 6숲 정반대 1등.

## 6. 피톤치드 지수 근거 (발표 방어용, 출처 고정)
국립산림과학원「산림치유자원 연구보고서」(전국 30지역 117지점, 2017~2024):
- 수종 순위: **편백>소나무>낙엽송>잣나무>활엽수** (p.13, p.88)
- 기온↑·습도↑·풍속↓ → 농도↑ (p.88, p<.01)
- 계절: 여름 최고, 단 **편백=3월·낙엽송=5월** 예외 (p.79~80)
- 시간: 6~9월 **정오 최저**, 야간 높음 (p.80)
- ※ 수종 '순위'는 보고서 근거, 점수 '간격'은 설계값. **상대 지수(0~100), 절대농도 아님.**

## 7. API 계약 (프론트 ↔ FastAPI)
**POST `/api/prescribe`**
```jsonc
// req
{ "goal": "수면", "lat": 37.55, "lon": 126.97,
  "user_type": "comfort", "month": 7, "hour": 10 }
// res
{ "pick": "잣향기 푸른숲", "score": 67.0,
  "visit_time": "오전 9~11시 (여름 정오 회피)",
  "why": "집에서 약 48km로 가깝고 ...",
  "target_outcome": "수면질 향상(주관 수면점수 +20%)",
  "post_measure": { "방문 전":["수면점수","피로도"], "방문 직후":["회복감","한줄기분"], "3일후":["수면점수"] },
  "ranking": [ { "name":"...", "total":67.0, "phyto":52.1, "distance_km":48 } ] }
```
**POST `/api/measure`** : 처방 id + 사전/사후 측정값 저장 → 효과 비교(데모는 더미 누적 시각화 가능).
※ 엔진 구현체 = prescription_engine.py (이 스펙대로 동작). FastAPI로 감싸면 됨.

## 9. 지금 작업 큐 (우선순위) — 데모 완성 3종
1. **38개 species 채우기**(이름 확보분 + 나머지 잠정) → 지수 실입력
2. **청정넷 n8n → 엔진 연결**(숲 최근접 관측소 매핑) → ③ 미세먼지 실시간화

4. (측정 루프) `/api/measure` + 더미 누적 효과 시각화
