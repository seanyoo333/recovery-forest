# 블로그 작성 가이드

이 가이드에서는 Supablog 시스템을 사용해서 효과적으로 블로그를 작성하는 방법을 설명합니다.

## 1. 기본 구조

모든 블로그 포스트는 다음과 같은 구조를 가져야 합니다:

```yaml
---
title: "포스트 제목"
description: "포스트 설명"
date: "YYYY-MM-DD"
category: "카테고리"
author: "작성자"
slug: "url-slug"
---
```

## 2. Markdown 작성

### 제목 사용

- `#` : H1 제목
- `##` : H2 제목
- `###` : H3 제목

### 코드 블록

```javascript
function hello() {
  console.log("Hello World!");
}
```

## 4. 이미지 추가

블로그 포스트에 이미지를 추가하려면:

1. `public/blog/{slug}/` 폴더에 이미지 파일을 업로드
2. 대표 이미지는 `cover.webp`, `cover.jpg`, `cover.png` 중 하나로 저장
3. frontmatter의 `image`에 대표 이미지 경로를 명시
4. 포스트에서 이미지 참조:

![블로그 작성 가이드](/blog/how-to-write-blog/cover.jpg)

## 5. SEO 최적화

- **제목**: 명확하고 검색 가능한 제목 사용
- **설명**: 포스트 내용을 잘 요약하는 설명 작성
- **카테고리**: 적절한 카테고리 선택
- **태그**: 필요시 태그 추가 (향후 기능)

## 6. 포스트 발행 후 확인

포스트를 작성한 후에는 다음을 확인하세요:

1. **로컬 개발 서버에서 확인**: `npm run dev`
2. **블로그 목록 페이지 확인**: `/blog`
3. **개별 포스트 페이지 확인**: `/blog/how-to-write-blog`
4. **소셜 미디어 공유 테스트**: Open Graph 이미지 확인

## 7. 모범 사례

### 좋은 제목의 예

- ❌ "새로운 기능"
- ✅ "React 18의 새로운 Suspense 기능 완벽 가이드"

### 좋은 설명의 예

- ❌ "React에 대한 글입니다"
- ✅ "React 18의 Suspense 기능을 실제 프로젝트에 적용하는 방법과 성능 최적화 팁을 다룹니다"

### 카테고리 선택

- **개발**: 기술 관련 포스트
- **가이드**: 튜토리얼, 가이드
- **업데이트**: 제품 업데이트
- **개인**: 개인적인 생각, 경험

이렇게 작성하면 독자들에게 유용하고 검색 가능한 블로그 포스트를 만들 수 있습니다!
