# PRD.md

## 1. 제품명

회복의 숲 (Recovery Forest)

## 2. 한 줄 정의

산림 공공데이터와 AI를 활용해 오늘 내 몸 상태와 관심사에 맞는 산림치유 공간(치유의숲·자연휴양림·도시숲·둘레길)과 산림치유 프로그램을 추천하는 서비스.

## 3. 출품 정보

- 대회: 2026 산림청 공공데이터 AI 창업 경진대회
- 마감: 2026-06-19 18:00
- 출품 주체: 에비던스 베이스 주식회사
- 서비스 도메인: `recovery-forest.evidence-base.ai`
- 현장 실증 파트너: 한국통합치유협동조합

## 4. 문제 정의

- 전국에 치유의숲·자연휴양림·도시숲·둘레길 등 산림치유 자원이 흩어져 있지만, 사용자가 "오늘 내 상태에 맞는 곳"을 고르는 방법이 없다.
- 미세먼지·기상·계절·체력·접근성·운동강도는 매일 바뀌지만, 사용자가 매번 비교하기 어렵다.
- 피톤치드 등 산림치유 효과 데이터는 공개되어 있어도 일반 사용자 언어로 정리되어 있지 않다.
- 산림치유 프로그램은 운영되고 있어도 검색·매칭이 어려워 참여율이 낮다.

## 5. 주요 사용자

### Primary Persona

회복기 중장년 (40~70대)

- 암 치료 이후 회복기, 만성질환 관리 중인 사용자
- 운동량 조절이 필요하고 무리한 트레킹은 어려움
- 미세먼지·기온에 민감, 면역·스트레스 관리 우선

### Secondary Persona

- 스트레스 관리가 필요한 직장인 (30~50대)
- 산림치유 프로그램에 관심 있는 가족 단위 방문자
- 통합치유협동조합 참여자, 산림복지 코디네이터

## 6. 제품 범위

### MVP 1순위 (대회 시연용)

1. **랜딩 페이지** — 서비스 소개 + 오늘의 추천 받기 CTA
2. **입력 폼** — 지역·건강 관심 우선순위·체력 수준·이동 가능 시간·선호 활동
3. **추천 결과** — 회복점수 기반 TOP 5 카드 (5축 점수, 추천 이유, 주의사항, 추천 활동)
4. **숲 상세** — 위치·트레일·예상 피톤치드·연계 프로그램

### 2순위 (대회 후)

- 결과에 대한 AI 후속 질문 (챗봇)
- 회원 가입 + "내 회복 숲" 저장
- 방문 후기/평점 수집
- 카카오톡 공유, 산림치유 프로그램 예약 연동

### 3순위 (장기)

- Evidence Base 본 플랫폼과 통합 (건강 리포트와 연결)
- 다국어 (영문 관광·웰니스 시장)
- 기업 임직원 산림치유 B2B

## 7. 범위 밖

초기에는 하지 않는다.

- 의료 진단 / 처방 / 치료 효과 단정
- 보조제·건강기능식품 추천
- 응급 의료 안내
- 실시간 GPS 기반 길찾기 (네이버/카카오 지도 위임)
- 호텔/숙박 예약

## 8. 추천 알고리즘 입력 차원

사용자 입력:
- 지역 (현재 위치 또는 시·군 선택)
- 건강 관심 우선순위 (다중 선택): 공기, 스트레스, 접근성, 면역력, 휴식, 편리성, 운동
- 체력 수준: 낮음 / 보통 / 높음
- 이동 가능 시간: 30분 / 1시간 / 2시간 이상
- 선호 활동: 산책 / 명상 / 체험프로그램 / 자유

실시간/공공 데이터:
- 기상청 단기예보 (온도·습도·풍속)
- 에어코리아 실시간 미세먼지 (PM10·PM2.5)
- 산림청 치유의숲 데이터
- 산림청 둘레길 / 자연휴양림 데이터
- 도시숲 데이터
- 4곳 치유의숲 실측 피톤치드 데이터 기반 예측 (수종·온도·습도·풍속)

## 9. 추천 점수 5축

| 축 | 의미 | 기여 데이터 |
|---|---|---|
| 공기 | 미세먼지·산소·피톤치드 | 에어코리아, 예측 피톤치드 |
| 기상 | 온도·습도·풍속의 회복 적합성 | 기상청 |
| 숲 | 수종·면적·산림밀도 | 산림청 |
| 운동 | 트레일 난이도·METs | 둘레길/숲길 데이터 |
| 접근성 | 대중교통·주차·이동 시간 | 거리 + 외부 데이터 |

총점은 사용자 우선순위 가중치를 곱한 가중합.

## 10. 제품 비전

단순 "어디 갈까" 추천 서비스가 아니다.

> 오늘의 내 몸 상태와 환경을 데이터로 읽고, 회복에 필요한 자연 자원을 매칭하는 산림복지 운영체제.

장기적으로 산림청·지자체·기업 산림복지 사업의 데이터 인프라가 된다.

## 11. 성공 지표

### 대회 시점 (2026-06-19)

- 시연 화면이 5개 이상 공공 API 호출을 시각화한다
- 5개 이상 우선순위 조합으로 모두 다른 결과가 나온다
- 추천 결과 카드가 평균 2초 이내 응답
- 발표자료에 평가항목 6개 (공공데이터·AI 기술·AI 서비스·독창성·발전가능성·ESG) 모두 매핑

### 정식 출시 후

- 추천 완료율 (입력 → 결과 도달)
- 결과 → 숲 상세 클릭률
- 산림치유 프로그램 연계 클릭률
- 재방문율
- 카카오톡/SNS 공유율

## 12. 사용자 경험 원칙

좋은 화면은 다음 질문에 답한다.

- 사용자가 여기서 무엇을 해야 하는가?
- 왜 이 숲이 추천되었는가?
- 무리하지 않는 가벼운 활동이 무엇인가?
- 오늘 가도 괜찮은가? (날씨·미세먼지)
- 가는 길이 멀지 않은가?

특히 40~70대 사용자가 모바일에서 한 화면씩 따라갈 수 있어야 한다.

## 13. 핵심 리스크

- **의료/광고법 표현 위반** — "치료" "효능" 단정 금지. 공공데이터 기반 안내 표현으로 통일.
- **공공 API 가용성** — 발표 도중 외부 API 다운. 결과는 Supabase에 영속 + 마지막 성공 결과 fallback.
- **AI 환각** — LLM이 데이터에 없는 시설/효능을 만들어냄. JSON 강제 + 안전 필터 + 데이터 ID 매칭.
- **6월 19일 마감** — 범위 확장 금지. MVP 1순위 외 기능은 모두 cut.
- **수상 후 운영 부담** — 1인 운영 가능한 자동화 우선, 사람 손이 많이 가는 기능 자제.

## 14. 제품 판단 기준

새 기능을 만들기 전 반드시 묻는다.

1. 회복 중인 사용자의 행동을 돕는가?
2. 공공데이터·AI 활용이 더 잘 보이는가?
3. 의료/법적 위험을 높이지 않는가?
4. 1인 운영자가 감당 가능한가?
5. 대회 평가항목에 매핑되는가?
6. Evidence Base 본 플랫폼 통합 시 자산이 되는가?

위 질문에 3개 이상 답하지 못하면 우선순위를 낮춘다.

## 15. 평가항목 매핑 (산림청 공모 기준)

| 평가항목 | 회복의 숲 대응 |
|---|---|
| 공공데이터 활용 | 5종 이상 (산림청 치유의숲/둘레길/도시숲, 기상청, 에어코리아) |
| AI 기술 활용 | LLM 기반 추천 이유 생성 + JSON 강제 + 안전 필터 |
| AI 서비스 완성도 | 입력→결과 전체 흐름 자동화 (n8n 오케스트레이션) |
| 독창성 | 4곳 치유의숲 피톤치드 실측 기반 전국 산림 예측 |
| 발전 가능성 | 산림청/지자체/기업 B2B + Evidence Base 통합 |
| ESG 혁신 | 산림 복지 격차 해소, 회복기 사용자 건강권 보장 |
