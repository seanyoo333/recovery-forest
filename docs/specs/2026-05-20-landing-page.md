# SPEC: 랜딩 페이지

## 1. 기능명

회복의 숲 랜딩 페이지 (`/`)

## 2. 배경

사용자가 처음 도착했을 때 "이 서비스가 무엇이며, 무엇을 할 수 있는지"를 3초 안에 이해하고 추천 받기로 이동해야 한다. 대회 심사위원도 첫 진입 시 이 화면을 본다.

## 3. 대상 사용자

- Primary: 회복기 중장년 (40~70대) 모바일 사용자
- Secondary: 대회 심사위원 데스크탑 사용자

## 4. 사용자 문제

- 전국에 숲은 많지만 오늘 내가 가도 좋은 곳을 모른다.
- 산림치유 정보가 흩어져 있다.
- 모바일에서 정보 비교가 어렵다.

## 5. 목표

- 3초 안에 서비스 가치 전달
- 1초 내 핵심 CTA 인지
- 데이터 출처 신뢰감 형성

## 6. 범위

### 포함
- 헤드라인 + 서브 카피
- "오늘의 회복 숲 찾기" CTA (`/recommend`로 이동)
- 서비스 동작 3단계 시각화 (입력 → 데이터 분석 → 추천)
- 데이터 출처 표기 (산림청·기상청·에어코리아 로고/텍스트)
- 한국통합치유협동조합 파트너 표시
- 푸터: 출품 정보, 이용 안내

### 제외
- 회원가입 / 로그인
- 블로그 / 콘텐츠 피드
- 검색 바
- 결제

## 7. 사용자 흐름

```text
1. 사용자가 `/` 진입
2. 헤드라인 + 서브 카피 1초 안에 인지
3. "오늘의 회복 숲 찾기" 버튼 발견
4. 버튼 클릭 → `/recommend` 이동
```

## 8. 입력 데이터

없음 (정적 페이지)

## 9. 출력/화면 결과

- 헤드라인: "오늘 내 몸에 맞는 회복의 숲을 찾다"
- 서브: "산림 공공데이터와 AI가 오늘의 미세먼지·기상·체력을 분석해 맞춤 숲을 추천합니다"
- 단일 CTA 버튼 (44px 이상, 컨트라스트 충분)
- 동작 흐름 3단계 카드
- 데이터 출처 섹션
- 푸터

## 10. 수용 기준

- [ ] `/` 경로에서 정상 렌더링
- [ ] 모바일 360px 폭에서 깨지지 않음
- [ ] CTA 버튼 클릭 시 `/recommend` 이동
- [ ] 첫 페인트 1.5초 이내 (Lighthouse 측정)
- [ ] 한국어 외 텍스트 없음 (영문 브랜드명 제외)
- [ ] 데이터 출처 5종 이상 표기
- [ ] 의료 단정 표현 없음

## 11. 오류/빈 상태

- Loading: skeleton hero (선택)
- Empty: 해당 없음 (정적)
- Error: 해당 없음
- Unauthorized: 해당 없음 (공개 페이지)

## 12. 보안/권한

- 공개 페이지, 인증 불필요
- 외부 트래커 미사용 (선택)
