# SPEC: 숲 상세 페이지

## 1. 기능명

산림치유 공간 상세 페이지 (`/forests/:forestId`)

## 2. 배경

추천 카드를 클릭한 사용자가 해당 숲의 자세한 정보(위치·트레일·예상 피톤치드·연계 프로그램)를 한 화면에서 보고 방문 결심을 할 수 있어야 한다.

## 3. 대상 사용자

- Primary: 추천 결과에서 1~5위 숲을 비교하는 사용자
- Secondary: 외부에서 직접 링크로 진입한 사용자

## 4. 사용자 문제

- 추천 카드만으로는 실제 방문 결심이 어렵다.
- 위치/접근 방법이 명확해야 간다.
- 산림치유 프로그램이 있다면 연계되어야 한다.

## 5. 목표

- 한 화면에서 방문 결심에 필요한 정보 모두 제공
- 네이버/카카오 지도 외부 링크
- 산림치유 프로그램 연계 (있는 경우)

## 6. 범위

### 포함

- 숲 기본 정보 (이름·종류·지역·면적·고도)
- 위치 표시 (좌표 → 외부 지도 링크)
- 수종 정보
- 트레일 정보 (길이, 난이도, METs)
- 예상 피톤치드 (4곳 데이터 기반 예측)
- 접근성 점수 + 대중교통/주차 안내
- 산림치유 프로그램 목록 (있다면)
- "다시 추천 받기" CTA
- 데이터 출처

### 제외

- 회원가입 / 즐겨찾기 (MVP 외)
- 결제 / 예약
- 후기 작성
- 실시간 지도 임베드 (외부 링크로 대체)

## 7. 사용자 흐름

```text
1. `/forests/:forestId` 진입
2. Loader가 Supabase `forest_places` 조회
3. healing_programs 조인하여 연계 프로그램 표시
4. 사용자가 외부 지도 버튼 클릭 → 네이버/카카오 지도 새 탭
5. 사용자가 산림치유 프로그램 클릭 → 외부 사이트 또는 안내 텍스트
6. "다시 추천 받기" → `/recommend`
```

## 8. 입력 데이터

- URL param: `forestId` (UUID)

## 9. 출력/화면 결과

- 헤더: 숲 이름 + 종류 뱃지
- hero: 이미지 (있으면) + 핵심 정보 카드
- 본문 섹션: 위치 / 트레일 / 피톤치드 / 접근성 / 프로그램
- footer: 데이터 출처

## 10. 수용 기준

- [ ] 유효 forestId → 상세 표시
- [ ] 무효 forestId → 404
- [ ] healing_programs 있으면 표시, 없으면 "연계 프로그램 정보가 없어요"
- [ ] 외부 지도 링크가 새 탭에서 열림
- [ ] 모바일 360px에서 깨지지 않음
- [ ] 데이터 출처 명시
- [ ] 의료 단정 표현 없음

## 11. 오류/빈 상태

- Loading: 본문 skeleton
- Empty (programs): "연계 프로그램 정보가 없어요" 안내
- Error: "정보를 가져오지 못했어요" + 결과 페이지로 돌아가기
- Unauthorized: 해당 없음

## 12. 보안/권한

- 공개 페이지, 인증 불필요
- Supabase RLS: forest_places anon read 허용

## 13. 의료/법적 안전 기준

- "치료" "효능" 단정 금지
- 산림청 데이터 표기로 신뢰 형성
